# 更新日志

## 2026-05-14

### 新增
- 新增 `CHANGELOG.md`，用于持续记录每次修改。
- 新增 Win7 双架构一键安装包脚本 `build_win7_dual_installers.ps1`：
  - 一次构建 Win7 x64 / x86 两个可执行文件；
  - 自动调用 Inno Setup 生成两个安装包；
  - 输出 `SHA256SUMS.txt` 校验文件。
- README 新增“双架构 Win7 安装包一键构建”使用说明。

### 变更
- 重点人模式时段评分逻辑调整为“类正态（钟形）”计分：
  - 以时段内出行次数计算时间分（0-40）；
  - 25 次为峰值最高分；
  - 5 次与 30 次作为两端低分；
  - 采用分段高斯处理左右不对称区间。
- 重点人模式“时段占比”口径改为：
  - 时段内出现次数 / 该车在本次处理（所选卡口）总出现次数。
- 重点人模式结果与导出文案统一：
  - `夜间占比` -> `时段占比`；
  - `夜间出现次数` -> `时段内出现次数`；
  - 结果页说明同步为新的评分规则。
- 重点人模式摘要统计增强：
  - 补充 `red/yellow/blue` 风险分布统计；
  - 新增“时段内活跃人数”摘要展示。

### 界面优化
- 去除筛选模式卡片前的单选圆框视觉（保留点击与选中逻辑）。
- 去除多选项卡片前的复选框视觉（保留点击与选中逻辑）。
- 增强 hover 态区分度（边框、背景、阴影更明显），提升可辨识性。
- 精简布局间距与提示文案，降低信息噪音。
- 移除多处不会实时更新的静态“已勾选 X 个/人/列”提示，仅保留实时状态行。

### 修复
- 修复 `build_win7_dual_installers.ps1` 调用 `build_exe.ps1` 的参数传递错误：
  - 原实现将参数以字符串数组转发，导致 `-Win7Compatible` 被误当作 `-PythonExe` 的值；
  - 现改为 hashtable splatting（`@{ Win7Compatible=$true; PythonExe=... }`）确保参数名和值正确绑定。
- 改进 Win7 双架构打包脚本的前置校验：
  - 在正式构建前即检查 x64/x86 两个 Python 路径是否存在；
  - 校验二者必须为 Python 3.8，且位数分别为 64 位 / 32 位；
  - 若 x86 环境缺失，直接输出可复制的创建命令，避免先打完 x64 再失败。
